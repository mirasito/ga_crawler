### Section 2: `## VPS setup from-scratch`

Ubuntu 24.04 LTS on Hetzner CX22 EU (Falkenstein/Helsinki). Setup in **exact order** (Pitfall #5 — `ga_crawler` user MUST exist BEFORE first logrotate run; Pitfall #6 — `useradd -m` для $HOME / Camoufox cache):

```bash
# 1. Install OS deps (cron + logrotate уже в base).
sudo apt update && sudo apt install -y curl sqlite3 logrotate cron git

# 2. Create system user (Pitfall #5 + Pitfall #6 — must precede logrotate + Camoufox).

# 3. Install uv (Astral) for the ga_crawler user.

sudo -u ga_crawler uv sync
sudo -u ga_crawler uv run playwright install firefox

# 5. Create log directory owned by ga_crawler.
sudo install -d -o ga_crawler -g ga_crawler -m 0755 /var/log/ga_crawler

# 6. Deploy cron + logrotate configs (Pitfall #1 — filenames have NO dots).
sudo cp deploy/etc-cron-d-ga_crawler /etc/cron.d/ga_crawler

sudo cp deploy/etc-logrotate-d-ga_crawler /etc/logrotate.d/ga_crawler
sudo chown root:root /etc/logrotate.d/ga_crawler && sudo chmod 0644 /etc/logrotate.d/ga_crawler
sudo systemctl reload cron

# 7. Populate .env (see §3).
sudo -u ga_crawler cp .env.example .env
sudo -u ga_crawler chmod 0600 .env   # T-07-08 mitigation
```

### Section 9: `## Логи`

Location: `/var/log/ga_crawler/weekly-run-YYYY-MM-DD.log[.gz]`.

Rotation: `/etc/logrotate.d/ga_crawler` (deploy/etc-logrotate-d-ga_crawler) — weekly + rotate 13 + compress + delaycompress. История 3 месяца. Hetzner CX22 40GB → ~5MB/run gzipped × 13 = ~65MB total (negligible).

Grep examples:

```bash
# Tail последнего run, все events:
tail -f /var/log/ga_crawler/weekly-run-$(date +%F).log | jq .

# Все errors последнего run:
grep '"level":"error"' /var/log/ga_crawler/weekly-run-$(date +%F).log | jq .

# History по run_id (включая gzipped archives):
zgrep '"run_id":42' /var/log/ga_crawler/*.log.gz | jq .

# Phase-level inspection:
grep '"phase":"goldapple"' /var/log/ga_crawler/weekly-run-$(date +%F).log | jq .
```

Edge cases:
- `.log.gz` archives появятся со 2-й недели (notifempty + delaycompress) — week 1's log rotates в первый Sunday после создания.
- 0-byte logs не ротируются (notifempty) — намеренно.
- Если weekly run extends past Monday 06:25 UTC (когда `/etc/cron.daily/logrotate` запускается на Hetzner EU) → wrapper может писать в rotated inode → bytes в `.log.1` (RESEARCH §Open Q #3 — acceptable edge для weekly batch).
