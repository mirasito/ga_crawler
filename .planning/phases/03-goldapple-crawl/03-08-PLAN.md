```python
def index_by_brand_token(
    slug_map: dict[str, list[str]],
    known_brand_tokens: set[str],
) -> dict[str, list[str]]:
    """Index URLs by brand-token prefix, picking the LONGEST prefix that is in
    known_brand_tokens.

    known_brand_tokens is the precomputed whitelist of all slug-variants of all
    viled brand aliases (computed via slug_fy_bilingual on every alias of every
    viled brand). The orchestrator builds this set BEFORE calling
    index_by_brand_token.

    For each product slug, we test depth-MAX..depth-1 prefixes (longest first)
    against known_brand_tokens. The URL is added to bucket[longest_matched_prefix]
    and ONLY that bucket. If no prefix matches, the URL is dropped (orphan slug —
    viled doesn't carry that brand, so we don't crawl it).

    This preserves D-305 / Pitfall 3 STRUCTURALLY:
    - 'tom-ford-beauty-eye-cream' with known={'tom-ford', 'tom-ford-beauty'} →
      depth-3 'tom-ford-beauty' matches first → bucket['tom-ford-beauty'] ONLY.
      bucket['tom-ford'] does NOT receive this URL.
    - 'tom-ford-noir-extreme' with same known → depth-3 'tom-ford-noir' NOT in
      whitelist; depth-2 'tom-ford' IS → bucket['tom-ford'] ONLY.
    - 'givenchy-pour-homme-blue-label' with known={'givenchy', ...} →
      depth-3 'givenchy-pour-homme' NOT; depth-2 'givenchy-pour' NOT;
      depth-1 'givenchy' IS → bucket['givenchy'].
    - 'jo-malone-london-cologne' with known={'jo-malone-london', ...} →
      depth-3 'jo-malone-london' IS → bucket['jo-malone-london'].

    Algorithm:
      tokens = slug.split('-')
      for k in range(min(MAX_DEPTH, len(tokens)), 0, -1):
          candidate = '-'.join(tokens[:k])
          if candidate in known_brand_tokens:
              bucket.setdefault(candidate, []).extend(urls)
              break  # ONLY add to longest-match bucket

    Time complexity: O(n × MAX_DEPTH) where n = len(slug_map). MAX_DEPTH = 3.
    """
```

    def index_by_brand_token(

    ) -> dict[str, list[str]]:
        """Index sitemap URLs by LONGEST brand-token prefix that is in known_brand_tokens.

        Path A from 03-VERIFICATION.md, longest-prefix-in-whitelist algorithm
        (Option 4). For each product slug:
          - test depth-3, depth-2, depth-1 prefixes (longest first)

          - 'jo-malone-london-cologne' with known={'jo-malone-london'}

          - 'unknown-brand-product' with known={'givenchy'}

        if not known_brand_tokens:

            if not slug:
