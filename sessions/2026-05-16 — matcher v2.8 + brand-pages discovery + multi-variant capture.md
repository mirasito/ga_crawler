## Ключевые архитектурные сдвиги

- **Discovery: sitemap → brand-pages.** `goldapple.kz/brands/{slug}` рендерит SPA → перехватываем `cards-list` XHR в браузере. См. [[Brand-pages discovery через cards-list + product-card API]].
- **Cards-list JSON содержит всё нужное** — itemId, brand, productType (русский!), name, attributes.units, price.actual. PDP-фетчи больше не нужны для основного flow.
- **Matcher v2.8 — три защитных слоя:**
  1. Volume-tolerant SQL (allow один-NULL)
  2. Russian product-type bucket veto (крем ≠ парфюм)
  3. VARIANT_MARKERS veto (extreme/refill/intense)
- **Multi-variant: cards-list даёт 1 itemId на семью.** [[cards-list возвращает 1 itemId на семью variant-ов]] — фикс через `product-card/base/v3?itemId=X` который отдаёт `variants[]` с per-variant ценой и itemId.
