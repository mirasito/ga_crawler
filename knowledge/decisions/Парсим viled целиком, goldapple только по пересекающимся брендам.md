# Парсим viled целиком, goldapple только по пересекающимся брендам

Отвергнуто: парсить goldapple целиком (десятки тысяч SKU). Принято: сначала viled → собрать список брендов → только эти бренды искать на goldapple.

## Почему

- **Цель отчёта** — сравнение цен на пересекающихся позициях. Бренды, которых нет на viled, нерелевантны.
- **Goldapple огромен** — полный парсинг = часы работы + большой расход residential-прокси-трафика
- **Снижение risk-surface** — меньше запросов = меньше шанс блокировки

## Следствия

1. Phase 2 (viled) **обязательно** идёт перед Phase 3 (goldapple) — viled даёт brand-list. См. ROADMAP.
2. На goldapple ходим по `brand` страницам, не по корневому каталогу
3. Brand-alias таблица — критично, иначе `Estée Lauder` на viled и `Эсте Лаудер` на goldapple не свяжутся. См. [[Brand-alias YAML — это v1 deliverable, не v2]]

## Открытые вопросы

- Существуют ли страницы вида `goldapple.kz/brand/<slug>`? Если нет — бренд-фильтр через search/category. Phase 1 спайк должен это выяснить. См. [[Текущие приоритеты — Phase 1 спайк]].
