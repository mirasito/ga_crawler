# Хранить полную историю снапшотов, не только текущий срез

Каждая неделя добавляет новый snapshot, помеченный `run_id`. Старые данные никогда не перезаписываются.

## Почему

- **Тренды цен** — команде нужна динамика, а не один срез
- **WoW-дельты** становятся тривиальным SQL-запросом (`run_id = current` vs `run_id = current - 1`)
- **Идемпотентность** — любую фазу пайплайна можно перезапустить для конкретного `run_id`
- **Recovery** — если что-то сломалось, есть прошлый рабочий снапшот для diff

## Хранилище

SQLite. См. [[БД — append-only snapshots с run_id]] для схемы.

## Стоимость

Десятки тысяч SKU × 52 недели × несколько лет = единицы GB. SQLite справится. Постгрес — в v2, если появится дашборд или multi-writer.

## Альтернатива (отвергнута)

"Хранить только последний срез + diff с прошлым" — экономит место, но теряет историю старше недели. Тренды стали бы невозможными.
